# COMPLETED-TASKS Skill

## Goal
Maintain a record of completed tasks for historical tracking, project metrics, and learning retention.

## Procedure
1. Review current HANDOFF.md to identify completed work
2. Extract key information about the completed task 
3. Add summary to _agent/COMPLETED_TASKS.md file with:
   - Task title and description
   - Completion date
   - Key outcomes achieved
   - Lessons learned
4. Update status in HANDOFF.md to mark as completed

## When to Use
- After completing any task that should be tracked for historical purposes
- When closing tasks that have significant impact or learning value

## Examples
- Recording completion of "Improve skill documentation and integrate template references" 
- Tracking the completion of "Update COMMANDS.md with accurate script listing"
- Documenting completion of "Add TODO file to track future project enhancements"

## Template Reference
This skill leverages the structure defined in `_templates/HANDOFF.template.md` which includes:
- Current status field
- What changed section 
- What still needs work
- Best next step guidance
- Files touched list
- Risks/warnings
- Suggested commands for verification
