# INDEX-SKILLS Skill

## Goal
Generate and maintain a comprehensive SKILLS-INDEX.md file in `_project/_skills/` that maps all available skills grouped by topic, making it easy for agents to discover and leverage the full skill set.

## Purpose
This skill creates a centralized index of all skills (core + project-specific) organized by functional categories, providing:
1. Quick discovery of relevant skills for specific tasks
2. Clear categorization of skills by domain/function
3. Direct references to skill locations and purposes
4. Updated documentation that agents can reference

## Procedure

### 1. Scan Skill Locations
- Read `.catalyst/_skills/` directory for core CATALYST skills
- Read `_project/_skills/` directory for project-specific skills
- Read `_agent/overlay/_skills/` if it exists (local overlays)

### 2. Extract Skill Metadata
For each skill directory, extract:
- Skill name (from directory name or SKILL.md frontmatter)
- Description (from SKILL.md frontmatter or first paragraph)
- Purpose/goal (from SKILL.md content)
- Location path

### 3. Categorize Skills
Group skills into logical categories:
- **Task Management**: PICKUP-TASK, CLOSE-TASK, UPDATE-TODO, CREATE-ACTIVE-TASK, COMPLETED-TASKS
- **Documentation**: IMPROVE-DOC, CONSOLIDATE-DOC, REVIEW-DOC, UPDATE-HUMAN-DOCS, UPDATE-README
- **Workflow & Planning**: WORK, PLAN, THINK, FEEDBACK, LEARN, ADAPT
- **Knowledge Management**: UPDATE-KNOWLEDGE-BASE, REMEMBER, DISTILL-LEARNINGS, PRUNE-KNOWLEDGE
- **Repository Maintenance**: BUILD-REPO-MAP, SYNC-QWEN-SKILL-REGISTRY, SYNC-COMMANDS, SYNC-BOOTSTRAP-FILES
- **Code Quality**: SAFE-REFACTOR, FIX-FAILING-TEST, IMPLEMENT-FEEDBACK
- **Catalyst Core**: DIGEST-LEGACY-CATALYST, RECONCILE-LEGACY-SKILLS, PROPOSE-CATALYST-CORE-CHANGE, PROPOSE-CATALYST-PULL-REQUEST
- **Feature Development**: ADD-FEATURE

### 4. Generate SKILLS-INDEX.md
Create/update `_project/_skills/SKILLS-INDEX.md` with:
```markdown
# Skills Index

## Overview
[Brief description of the skills ecosystem]

## Quick Reference Table
| Category | Skill Count | Description |
|----------|-------------|-------------|
| ... | ... | ... |

## Detailed Categories

### Task Management
- **[PICKUP-TASK]** - Selects and converts a task from the TODO list into an active task for execution.
  - Location: `.catalyst/_skills/PICKUP-TASK/SKILL.md`
  
- **[CLOSE-TASK]** - Closes or rolls over the active task by updating handoff, memory, and task state.
  - Location: `.catalyst/_skills/CLOSE-TASK/SKILL.md`

### Documentation
...

## Usage Guidelines
[Best practices for selecting and using skills]

## Maintenance
This index is auto-generated by the INDEX-SKILLS skill. Run this skill when adding new skills or updating existing ones.
```

### 5. Update Agent Documentation
Update relevant agent documentation to reference the index:
- Add link to SKILLS-INDEX.md in `_agent/state/SCOPE.md`
- Add link in `_agent/state/ACTIVE_TASK.md` template if applicable
- Mention in `AGENTS.md` under "Available Skills" section

## When to Use
- After adding new skills to the repository
- When agents need to discover available capabilities
- During project setup or onboarding
- Periodically to ensure index accuracy

## Output Files
1. `_project/_skills/SKILLS-INDEX.md` - The main skills index (created/updated)
2. Updated agent documentation files referencing the index

## Constraints
- Preserve existing skill descriptions and metadata
- Maintain consistent formatting across all entries
- Use relative paths for skill locations
- Keep categories logical and intuitive
- Ensure markdown links are valid and navigable

## Example Usage
```bash
# Run the INDEX-SKILLS skill to generate/update the index
skill: "INDEX-SKILLS"
```
