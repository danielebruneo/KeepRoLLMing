## Summary of Changes

### New Features Added to `perf_dashboard.py`

1. **Interactive Key Bindings**
   - `q`: Quit dashboard (alternative to Ctrl+C)
   - `c`: Clear logs directory (`__perf_logs`)
   - `s`: Save timestamped backup of `summary.yaml`

2. **Terminal Mode Handling**
   - Implemented non-blocking key capture using `termios` and `select`
   - Properly saves/restores terminal settings to maintain Ctrl+C functionality
   - Uses raw mode only during key reading, not during rendering

3. **Data State Management**
   - Clear cached models when logs are cleared
   - Immediate display of "<no data>" state after reset
   - Prevents stale data from persisting in UI

4. **Backup System**
   - Saves to `__perf_logs/backups/` directory
   - Timestamped filenames: `summary_YYYYMMDD_HHMMSS.yaml`
   - Uses `shutil.copy2` to preserve metadata

### Technical Decisions & Lessons Learned

#### Why Raw Mode in Background Thread Didn't Work
**Problem:** Setting terminal mode in a background thread conflicted with main loop's keyboard interrupt handling.

**Solution:** Applied raw mode temporarily within the main loop, saved original settings, and restored them in `finally` block. This ensures:
- Ctrl+C still works (terminal signals aren't disabled)
- Key presses are captured immediately without waiting for Enter
- Terminal returns to normal state on exit

#### Why `sys.stdout.flush()` Was Removed
**Problem:** Added flush calls caused display issues with line breaks not appearing correctly.

**Solution:** Rely on Python's default buffering behavior with `print()`. The terminal handles line buffering appropriately for interactive applications.

#### Key Capture Strategy
**Initial Attempt:** `sys.stdin.read(1)` in background thread - didn't work in canonical mode (terminal buffers input).

**Final Approach:** `os.read(fd, 1)` with `select.select()` timeout:
- Works in non-canonical mode (ICANON disabled)
- Non-blocking with short timeout (0.05s)
- Doesn't interfere with main render loop

## Known Limitations & Future Improvements

1. **Raw Mode Side Effects:** Terminal cursor behavior may appear unusual during key capture (no echo, no line buffering)
2. **No Confirmation Dialog:** Clearing logs happens immediately - consider adding confirmation prompt for safety
3. **Single Dashboard Instance:** Multiple dashboard instances could conflict when writing to same summary file
4. **No Undo:** Once logs are cleared, they're gone - backup feature helps but could be more prominent

## Recommendations

### For Users
- Use `s` key regularly to create backups before major testing sessions
- Remember that `c` clears ALL data in the logs directory
- Dashboard will show "<no data>" until new performance metrics are generated

### For Developers
- If adding more key bindings, consider a menu system instead of single characters
- Consider using a proper TUI library (textual, rich) for better terminal handling
- Add keyboard shortcut hints to footer that update based on context
