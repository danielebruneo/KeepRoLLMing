## Key Insights Learned
1. **CATALYST File Structure Understanding**:
   - Agent state files (`_agent/state/`) are runtime/temporary (ACTIVE_TASK.md, SCOPE.md, etc.)
   - Project documentation (`_project/`) contains permanent knowledge about the project structure 
   - The `_agent/knowledge/` directory is for durable knowledge artifacts

2. **Documentation Location Patterns**:
   - `MAP.md` files contain repository structure information
   - `COMMANDS.md` should be in `_project/` not `_agent/state/`
   - `_agent/MEMORY.md` contains lessons learned from working on the project
   - `_agent/HANDOFF.md` documents handoff status

3. **Skill Execution Best Practices**:
   - UPDATE-KNOWLEDGE-BASE skill refreshes existing knowledge files, not creates new ones
   - Before implementing changes, always verify what already exists in the repository 
   - Proper distinction between runtime state and permanent documentation is crucial for CATALYST workflows

## Pattern Recognition
- When updating knowledge base components, I should first check if files exist before creating them
- Project commands belong in `_project/COMMANDS.md` while agent runtime information belongs in `_agent/state/`
- The skill process involves identifying gaps rather than creating everything from scratch
