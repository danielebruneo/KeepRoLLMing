# UPDATE-README Skill

## Goal
Update [README.md](CATALYST.md) without disturbing stable sections unnecessarily.

## Procedure
1. Read the current [README.md](CATALYST.md) fully - understand what's already there.
2. Identify what actually changed in the project - focus on material modifications.
3. Update only the sections impacted by the change - avoid touching unchanged areas.
4. Keep the README human-oriented - make it accessible to users and developers.
5. Avoid injecting task-local implementation notes - keep it focused on usage.

## When to Use
- When project structure or functionality changes significantly
- When adding new features or capabilities that should be documented
- When updating installation or usage instructions

## Examples
- Adding new configuration options to README documentation
- Updating usage examples after feature additions
- Modifying setup instructions when environment changes
