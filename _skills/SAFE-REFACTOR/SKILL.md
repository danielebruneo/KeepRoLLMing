# SAFE-REFACTOR Skill

## Goal
Improve internal structure without changing intended behavior.

## Procedure
1. Define the refactor scope clearly.
2. Identify behavior that must remain unchanged.
3. Prefer incremental, reviewable changes.
4. Validate with the most relevant checks.
5. Stop if the work starts turning into a redesign.
