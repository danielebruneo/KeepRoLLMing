# Style Guide

Use this file for human-facing coding conventions.

## Suggested topics
- naming
- file organization
- error handling
- logging style
- test style
- documentation style

Agent-only operating rules belong in `_agent/CONSTRAINTS.md`, not here.

## Documentation Formatting Standards
All documentation files should follow consistent formatting:
1. Use markdown headers with appropriate hierarchy (H1, H2, H3)
2. Maintain consistent section naming patterns
3. Include cross-references to related documents and templates
4. Follow [ACTIVE_TASK.template.md](../_templates/ACTIVE_TASK.template.md) for new documentation elements

## Template Integration
Documentation files should explicitly reference the template used:
- Add "Template Reference" sections that link to relevant templates
- Include "Cross-referencing" sections listing other related documents
