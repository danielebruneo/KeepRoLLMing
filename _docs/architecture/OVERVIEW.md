# Architecture Overview

This file is for humans.

## What belongs here
- Project purpose
- High-level architecture
- Main components
- Boundaries and responsibilities
- Major request/data flow

## What does not belong here
- Session-specific notes
- Task status
- Agent operating instructions
- Temporary debugging notes

## Template
### Project purpose
What problem the project solves.

### Boundary
What the project is and is not responsible for.

### Main components
- Component A
- Component B
- Component C

### Request / data flow
A concise description of how work moves through the system.

### Failure / fallback philosophy
How the system degrades gracefully.
