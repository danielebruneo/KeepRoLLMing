# IMPROVE-AGENT-WORKFLOW Skill

## Goal
Enhance the complete agent workflow system to improve consistency, usability, and integration between project components.

## Procedure
1. Review current agent workflow patterns - examine how skills interact with documentation files.
2. Identify areas where template-based systems can be better leveraged for consistent formatting.
3. Ensure datetime tracking is applied across all relevant files and workflows.
4. Improve cross-referencing between knowledge base, documentation, and skill components.
5. Standardize integration patterns to reduce confusion and improve usability.
6. Validate that workflow improvements maintain backward compatibility.

## When to Use
- When improving the overall agent project workflow system 
- When seeking to enhance consistency across all documentation and skills
- After implementing multiple related improvements to consolidate them into a unified approach
- When maintaining or upgrading agent knowledge base components

## Examples
- Integrating template-based workflows consistently across all project files
- Applying datetime tracking uniformly throughout the agent system  
- Improving cross-referencing between documentation and skill implementations
- Standardizing formatting and structure for better navigation and understanding

## Workflow Enhancement Features
This skill implements comprehensive improvements including:
- Template-based system integration (ACTIVE_TASK.template.md, HANDOFF.template.md)
- DateTime tracking across all workflow files (DD/MM/YYYY HH:MM:SS format)
- Documentation reference guidance for agent use
- Consistent formatting patterns throughout the project
- Enhanced cross-referencing between documentation components

## Integration Points
This skill enhances integration between:
- Skills and knowledge base files (_agent/MAP.md, _agent/COMMANDS.md)
- Skills and documentation files (_docs/*) 
- Templates and actual implementation workflows
- All agent workflow components for consistent usage patterns
