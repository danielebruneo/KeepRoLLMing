# SKILL: UPDATE-README-SAFE

## Goal
Update the public README without damaging its human-oriented role.

## Procedure
1. Read the current `@README.md` fully - understand existing content.
2. Identify the exact outdated section - pinpoint what needs updating.
3. Update only that section unless broader change is requested - avoid unnecessary modifications.
4. Keep tone consistent - maintain the same writing style and approach.
5. Do not inject internal agent workflow into public docs unless explicitly desired - focus on user needs.

## When to Use
- When making targeted updates to specific README sections
- When fixing outdated information without major restructuring
- When improving clarity or accuracy of existing documentation

## Examples
- Updating a single configuration parameter description in README
- Correcting an outdated usage example
- Improving formatting or spelling in existing documentation
