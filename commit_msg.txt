Fix prompt_tps calculation for streaming and non-streaming requests

- Fixed performance.py: Changed prompt_tps formula from `prompt / ((ttft + (elapsed - ttft)) / 1000)` 
  which simplified to elapsed time, to correct `prompt / (ttft / 1000)`
  
- Added TTFT estimation for non-streaming requests in app.py (~30% of total elapsed time)
  since we can't measure actual first-token arrival without streaming
  
- Added prompt_tokens estimation for streaming requests when API doesn't provide them
  (completion_tokens * 0.3 ratio), enabling prompt_tps calculation

- Fixed healthcheck.py to use resolve_inherited_route() instead of resolve_route() 
  for proper route inheritance resolution without pattern matching
